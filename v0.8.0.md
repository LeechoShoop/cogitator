# Cogitator v0.8.0

Cogitator is a TLS-intercepting proxy and pentest toolkit written in Rust,
with a terminal UI — same general space as Burp Suite, built from scratch
as a Cargo workspace. This is the first tagged release.

## Highlights

- **Full traffic interception**: HTTP/1.1 and HTTP/2 (independent per-leg
  ALPN negotiation), plus complete WebSocket interception with live
  Forward/Drop/Replace on individual frames.
- **Active scanner**: SQLi (error-based + time-based blind), path
  traversal, context-aware XSS, SSRF (with optional OOB DNS
  confirmation), and XXE — async, concurrency-capped, severity-sorted.
- **Distributed scanning**: fan a scan out across any number of
  `cogitator-worker` processes on other hosts.
- **Plugin system**: write checks as external `.so`/`.dll` plugins
  against a versioned, ABI-checked shared crate, or ship built-ins via
  `inventory`.
- **Reporting**: HTML and paginated PDF scan reports, with finding
  status tracking (New/Confirmed/False Positive/Fixed).
- **Security-conscious by default**: CA private key encryption, an
  encrypted session vault, and log redaction that strips auth
  headers/cookies before anything hits disk.

See `CHANGELOG.md` for the full breakdown by subsystem.

## Getting started

```bash
cargo run
```

The first run generates a local CA (`cogitator_ca.pem` /
`cogitator_ca.key`) — import the `.pem` into your browser or OS trust
store to intercept TLS traffic without certificate warnings. Read
`SECURITY.md` before pointing this at anything with real credentials in
it: the default auto-save file is plaintext, and the CA key can
impersonate any host to any device that trusts it.

For distributed scanning, see `distributed_scanning_setup.md`. For
writing a plugin, see `PLUGINS.md`.

## Known limitations in this release

- Auto-save on exit (`cogitator_last.cogitator`) is plaintext; use
  `Workspace-Save` with a passphrase if the captured traffic is
  sensitive.
- Distributed scanning v1 auth is a single shared bearer token over
  plain HTTP — no per-worker credentials, no TLS, no retry/health-check
  before dispatch. Treat it as a same-trust-boundary feature (e.g. your
  own LAN or VPN), not something to expose publicly.
- No built-in authorization/consent verification. Cogitator is built for
  systems you own or are explicitly authorized to test — see
  `SECURITY.md` §3 for the full scope statement.
